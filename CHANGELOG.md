# Changelog

## v5 — Полный аудит и фиксы (2026-04-23)

По результатам сквозного аудита кода, API и UI:

### Надёжность NASA API (`lib/safeFetch.ts`)
- Добавлены повторные попытки (`retries=2` по умолчанию) с экспоненциальным backoff (400 → 800 → 1600 мс).
- Таймаут снижен с 15 с до 8 с на попытку — быстрее отдаём ошибку/ретраем.
- Встроен in-process кэш с TTL; при падении апстрима серверного рендера/route handler отдаём stale-значение вместо 503.
- Умный retry: на 5xx/429/403 — повторяем, на 4xx клиентских — нет.
- `api_key` скрывается в логах (`api_key=***`).

### APOD карточка (`components/nasa/NasaApodCard.tsx`)
- Кнопка «Попробовать снова» (`refetch`) — пользователь больше не заперт при временной ошибке.
- Более понятный текст ошибки (не обвиняет `DEMO_KEY` по умолчанию).
- React Query: `retry=2`, `staleTime=1h`.

### Сцена — корректный cursor при hover
- `PlanetMesh` и `GlobeMarker`: переход с глобальной мутации `document.body.style.cursor` на `useCursor()` из `@react-three/drei`. Устраняет «залипший» указатель при анмаунте сцены (например, когда открывается modal планеты) и убирает предупреждения React о сторонних эффектах в render-фазе.

### Фото / контент
- Футер: упоминание Wikimedia заменено на актуальные источники — Solar System Scope (CC BY 4.0) и NASA Visible Earth.
- Подтверждено: все 7 фото архитектуры отдаются из `/public/architecture/*.jpg` (200 OK).
- Подтверждено: все 15 текстур в `/public/textures/` грузятся (200 OK).

### Проверки
- `npm run lint` → 0 ошибок
- `npm run typecheck` → 0 ошибок
- `npm run build` → 7 маршрутов, 0 ошибок
- Live проверка в браузере: Console без ошибок, APOD, EPIC (8 снимков), Mars (9+ фото c Curiosity), DONKI (Солнце), все вкладки Земли (Факты / Архитектура / 3D-глобус / NASA), клики по Солнцу, Земле, Марсу — работают.
